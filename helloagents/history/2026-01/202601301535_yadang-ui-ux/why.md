# 变更提案: yadang-1 UI/UX 复刻（Next 16 App Router）

## 需求背景
当前项目仍为默认 Next App Router 模板。需要以 `yadang-1` 目录为 UI/UX 参考，在本项目中实现同等视觉与交互，并完成工程化目录组织与 API 统一封装；当前阶段只做前端 UI/UX，后端接口以伪代码/Mock 形式提供。

## 产品分析

### 目标用户与场景
- **用户群体:** 知识产权相关专业人员、设计师、企业法务
- **使用场景:** 上传图片并快速比对相似外观专利/商标
- **核心痛点:** 查询入口分散、比对效率低、结果可信度与展示体验不足

### 价值主张与成功指标
- **价值主张:** 用视觉检索提升 IP 检索效率，提供专业、可信的产品体验
- **成功指标:** 首屏可用率、查询流程完成率、关键区域复刻一致性

### 人文关怀
用户上传素材涉及商业机密，需避免过度上传与不必要的状态泄露；本期仅本地保存历史记录，不做真实传输。

## 变更内容
1. 采用 Route Groups 组织路由，预留未来 admin 扩展
2. 使用 shadcn/ui 组件重构 UI，与 yadang-1 的视觉一致
3. 统一 API 请求封装与 Mock 逻辑
4. 使用中文文案，暂不引入国际化
5. 全局样式与动画体系与 yadang-1 对齐并做性能约束

## 影响范围
- **模块:** app、components、lib、hooks、styles
- **文件:** app/layout.tsx、app/globals.css、components/*、lib/*、hooks/*、可能新增 stores/*
- **API:** 新增伪接口封装（auth/search/recharge/history）
- **数据:** 本地状态与 localStorage 持久化

## 核心场景

### 需求: 首页视觉与交互复刻
**模块:** app/components
首页视觉需与 yadang-1 对齐

#### 场景: 用户进入首页
无
- 展示 Header、主视觉区、上传区、配置区、历史侧边栏
- 动效、阴影、渐变与布局层次与参考一致

### 需求: 上传检索与结果展示
**模块:** components
支持拖拽上传、预览、配置切换与结果列表

#### 场景: 上传并检索
用户选择图片与配置
- 展示加载态与扫描动画
- 返回 mock 结果并可切换结果标签

### 需求: 详情弹窗与认证/充值
**模块:** components
统一使用 Dialog 组件

#### 场景: 打开详情弹窗
用户点击结果卡片
- 展示详情信息与操作按钮

#### 场景: 登录/注册/充值
用户点击登录或余额不足
- 展示对应弹窗与表单校验

### 需求: 历史侧边栏
**模块:** components
展示最近搜索历史与快速回填

#### 场景: 点击历史记录
用户点击历史项
- 自动回填配置与结果列表

### 需求: API 统一封装
**模块:** lib
所有请求统一走 Client/Service

#### 场景: 触发检索请求
上传图片后调用 API
- 返回稳定的 mock 结构以供 UI 使用

## 风险评估
- **风险:** 过量模糊/大阴影/复杂动画导致 GPU 压力
- **缓解:** 控制模糊半径与层级、减少大面积 blur、提供 reduced-motion 样式
